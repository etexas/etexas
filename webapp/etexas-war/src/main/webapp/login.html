eTEXAS

Enhanced Traffic EXperimental Analytical Simulation

eTEXAS is a platform for connected vehicle application simulation. This empowers researchers and accelerates the maturity of Connected Vehicle Program applications. We accomplish this by leveraging the modeling detail in simulation models, naturalistic vehicle trajectory datasets and message transmission modeling. We then provide this to researchers and organizations building Connected Vehicle Program applications for a variety of objectives including safety, mobility, and ecological concerns.

eTEXAS follows Connected Vehicle standards such as SAE J2735 messages, which includes:

  • SPAT (signal phase and timing) messages that report traffic signal status
  • MapData messages that define intersection geometry
  • BSM part 1 and 2 (basic safety messages) messages that report vehicle movement
  • CSR (common safety request) messages that request additional vehicle information

eTEXAS allows Connected Vehicle Apps to:

  • Send and receive DSRC/WAVE messages (e.g., J2735)
  • Access data that would be available to them through DSRC units installed on either vehicles or road-side units (i.e., simulated vehicle CAN-bus data, simulated traffic signal data, simulated lane geometry, etc.)
  • Influence the simulation on-the-fly through a command interface (i.e., vehicle commands that allow apps to change a simulated vehicle’s speed or signal commands that allow apps to change a simulated signal’s phase or timing)

eTEXAS adds a wireless network simulation model to traditional traffic simulation models for the following modeling advantages:

  • Broadcast/multicast/unicast message routing
  • Transmission latency
  • Propagation/fading
  • Message collision
