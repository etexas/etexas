Using the Execution Visualization

Use the View toggle on the Execution Details toolbar to open the visualization for an execution. The intersection(s), and objects you configured in Settings (for the composite and/or the simulations) are displayed in an interactive interface.

When you begin advancing the execution, additional objects are added to the display, such as vehicles, detectors, and signals.

These buttons appear at the top of the Visualization, below the Display banner:

  • View allows you to choose which components to include in the visualization; all are on by default.
  • Camera View opens a selection list of the Simulations contained in the composite. Select the simulation you want to view. This is a quick way to re-center the visualization on the selected simulation when it has scrolled off the screen.

In the image below, the Heads-Up Display (HUD) of orientation has been disabled in the View menu. The View options can help to de-clutter the screen.

Resize (drag the vertical blue bar) or hide (arrow button in panel heading, see image below) the Executions and Display Info panes in the left and right sides of the window to give more screen space to the visualization.

Orientation and Position

The initial view in a visualization is an almost-flat diagram, as if you were viewing an intersection from directly overhead. It is centered on the 0,0 location (which is often used for the location of the first intersection in a composite). Use Camera View, scroll, and/or elevation to navigate in this view (see the table below for keyboard commands).

The orientation graph in the upper right corner shows the relative position of the X, Y, and Z coordinates that are used for placing objects such as buildings, RSE, etc. As you rotate and tilt the display, the graph moves to reflect the current position of the intersection.

In the example below, a building is in the lower right. When you click an object, the pane at the right side of the screen displays information about the object (here, the building). Notice that at this orientation, the positive X (7000 cm) and Y (4000 cm) values place the building in the lower right quadrant.

 

The image below is the same visualization as above, but note how the orientation control has changed to show the position of each axis, and the building, RSE, and fixed cellular device are in different areas of the screen.

Display Information Pane

Click an object in the visualization to get more information about the object. For stationary objects, this information includes size and position. For vehicles, the information is current, so it may be lane number, position, speed, devices on board, etc.

This information can be viewed only while the execution is in progress.

Also see Transmission Path and Radius for viewing message paths and broadcast radius.
