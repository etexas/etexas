Acronyms, Terms, and Common Elements

Help Features

This help system uses several common features and styles, such as the appearance of button text. These items, and a list of acronyms used throughout Help, are shown here.

Most large images in Help display as thumbnails, until you hover over them. Try it with the image below:

Search Bar - The search bar always appears at the top of the Help window. Enter a word or phrase to search for, and press Enter or click the search button.

Text Styles in Help

button - used for all button names in the interface

menu or selection - used for menu and sub-menu names, and names of perspectives

field name - used for field names and table column headings

command - used for commands to be typed by a user

Common Interface Elements

Throughout the eTEXAS interface, you will see these features:

Feature / Icon Description
Help icon; opens context sensitive help.
Resize pane. Several perspectives have multiple panes; put the mouse pointer on the dividing line and click-and-drag to resize.

Select multiple items in a list

In any view containing a list, such as composites, simulations, buildings (and other settings), hold the Ctrl key and click items to select. If you are selecting several contiguous items, you can also select the first item, hold Shift, and click the last item. These selection keys are useful if you are deleting multiple items at once.

When more than one item is selected, options that apply to only a single item are grayed out. For example, when you select multiple Composites, the Edit button is disabled.

Acronyms and Terms

Acronym or Term Definition

DSRC

Dedicated Short Range Communications

eTEXAS

Enhanced Traffic Experimental Analytical Simulation

ID

Identification (number). Most components in eTEXAS are automatically assigned an ID number. In table views, such as the lists of composites and simulations, and the execution text view, the ID appears in the first column. In a visualization, the ID of components such as vehicles, RSE, cell towers, etc. appears on the component.

JAR

Java Archive

OBU

On Board Unit

Perspective

Primary views within eTEXAS; switch perspectives by clicking the names in the green banner (Applications, Executions, Simulations)

RSE

Road Side Equipment

Rx

Rx denotes messages received by a selected device

SPAT

Signal Phase and Timing

Tx

Tx denotes messages transmitted (sent) by a selected device

X, Y, Z coordinates

Measurements used to define, in centimeters, the locations of intersections (X, Y) and fixed-position items (such as RSE) relative to the center of a composite.

Initially, the visualization view of an execution shows X as vertical or top-bottom; Y is horizontal, or left-right; Z is elevation. However, in an execution, you can change the position, or orientation, at which you are viewing an intersection; a heads-up display on the screen rotates to show the relative position of X, Y, and Z coordinates to the current view.

When specifying the location of an object (such as a cell tower or RSE), the X, Y, Z coordinates are the center of the object; for a building it is the center of the base of the building.
